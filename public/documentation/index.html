1.Introduction

ReactJS is the latest JavaScript framework that forces you to think regarding components. This model of thinking fits user interfaces well. Depending on your background it might feel alien at first. You will have to think very carefully about the concept of state.

2.File Structure

Webmin React file structure in the webmin-react folder.

Below is a list of the file structure in a react app.

Path Description
-src Main source folder.
         -assets Application assets such as a custom css , js , images , scss.
--css Compiled custom css files.
--image Images used for the application.
--js Third parties javascript files.
--scss Module wise scss files.
    -components  All components source folder.
--authentication Authentication is used for authentic the users.
    -login Login module Implement with firebase.
    -registration Registration module implement with firebase.
--calendar Calendar module is used for full page calendar.
--chart Chart module is used for different type of chart.
--chat Chat module for a chat with the users.
--custompages Custom pages contain multiple pages like Blank page, Contact us, Error, Faq and Invoice page.
--dashboard Dashboard of an application.
--elements UI kit elements for entire the application.
--firebase Firebase is used for the configuration web application with firebase project.
--fonticon Font icon folder contains font awesome icons.
--forms Forms module such as Form Elements, Form Group, Form Validation etc.
--layout Layout folder is used to set the layout of an application.
--mailbox Mailbox design module used for mailing application.
--maps Maps module use for display full-page map.
--tables Used for a basic table and data table are used.
--widgets Different types of data widgets module for the application.
--app.css Main application stylesheet file.
--app.js Main application js File
--index.html The index.html lives at the root of the front-end structure.
--index.js Whenever an application is run, First  index.js file is a load.
--router.js Base routing for the application.
--validation.js Application in used all validation load into this file.
--vendor.js Application in all CSS and JS load into this file.
package.json A package.json file contains metadata about app or module. Most importantly, it includes the list of dependencies.
webpack.config.js Webpack is a module bundle which takes modules with dependencies and generates static assets by bundling them together based on configuration.
webpack.dev.js This file is used for development purpose and will run when the modification is going to be done on localhost.
webpack.prod.js This file is used for production purpose when the build command is fired to run the file on server.

3.Installation

This section guides you to install all required software and libraries to utilize webmin  for your projects fully.

3.1.Node

Node.js® is a JavaScript runtime built on Chrome’s V8 JavaScript engine. Node.js is required to build the Webmin React. Download the latest version of Node and install it: Download Nodejs

3.2.NPM

Npm is the package manager for JavaScript and the world’s largest software registry. Npm is a separate project from Node.js and tends to update more frequently. As a result, even if you’ve just downloaded Node.js (and therefore npm), you’ll probably need to update your npm.

 npm install --global npm@latest

Verify that npm in successfully installed, and version of installed npm will appear

 npm --version

3.3.React

Installation

 

1. First, unzip the zip file that you have download from ThemeForest. You will find a webmin-react project and readme file.

2. Then after Extract zip File (webmin-react.zip) into a folder, you have to work within.

3. Open a Node Prompt. Go to your Working Folder, And run the Following Command and wait for finish installation.

npm install

4. This command will install all the required node module into node_module/ inside your working folder.

5. You are ready to run the react-webmin application for the first time

 

Working With React-Webmin

 

While still your working folder And run the following command

npm start
